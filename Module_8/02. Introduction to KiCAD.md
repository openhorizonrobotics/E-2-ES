KiCAD is a free, open-source electronics design automation (EDA) suite used by hobbyists and professionals alike. It gives you everything you need to go from a circuit idea to a set of fabrication-ready files -- without a licensing fee or a cloud subscription. It runs on Windows, macOS, and Linux, and for a tool of its capability, it has a surprisingly shallow learning curve once you understand its structure.

You can download the latest version from the official KiCAD website: [https://www.kicad.org/download/](https://www.kicad.org/download/)

#### How KiCAD Thinks About a PCB Project

Before touching any tools, it helps to understand how KiCAD organises its work. A KiCAD project is not a single file. It's a folder containing several files that each describe a different aspect of your board.

| File            | What it contains                   |
| --------------- | ---------------------------------- |
| `.kicad_pro`    | Project settings and configuration |
| `.kicad_sch`    | Your schematic                     |
| `.kicad_pcb`    | Your PCB layout                    |
| `sym-lib-table` | Symbol library references          |
| `fp-lib-table`  | Footprint library references       |

The schematic and the PCB layout are separate documents that are linked together. You design the circuit logically in the schematic first, then push that information into the PCB editor where you arrange and route everything physically. This separation is deliberate and important. It lets you think about _what connects to what_ before worrying about _where everything physically sits_.

Keep this mental model in mind: **schematic = logical, PCB = physical.**

#### The Schematic Editor

The Schematic Editor is where your circuit takes shape as a logical diagram. Think of it as a formal, machine-readable version of the hand-drawn circuit sketches you may already be familiar with. Every component is represented by a **symbol** -- an abstract drawing that shows the component's pins and their functions, without any reference to physical size or shape.

##### Symbols and the Symbol Library

A symbol is just a drawing with named, numbered pins. KiCAD ships with an extensive built-in library covering resistors, capacitors, microcontrollers, op-amps, connectors, and much more. You place symbols from the library using `Place → Add Symbol` or by pressing `A`.

![](Module_8/Assets/5.png)

*Figure: Symbol chooser dialog with search bar*

Most common components are already in the library. For anything unusual -- a specific IC, a breakout board connector, or a custom module -- you may need to create your own symbol or import one. Many component manufacturers and vendors like SnapEDA or Ultra Librarian provide KiCAD-compatible symbols for download.

- Find SnapEDA (renamed to SnapMagic) here: https://www.snapeda.com/home/
- Find Ultra Librarian here: https://www.ultralibrarian.com/cad-vendors/kicad/
##### Nets and Wires

Once your symbols are placed, you connect their pins with **wires** (`Place → Wire`, or press `W`). Each continuous electrical connection in your schematic is called a **net**. Nets are the foundation of everything -- they define what connects to what, and KiCAD tracks them throughout the entire design flow.

For connections that would make the schematic visually cluttered if drawn as explicit wires (power rails like VCC and GND being the most common) KiCAD provides **power symbols** and **net labels**. Any two pins sharing the same net label are electrically connected, even if no wire visibly joins them.

![](Module_8/Assets/7.png)

*Figure: Notice how none of the nets are physically connected. The labels implicitly connect them.* 

##### The Rationale for a Clean Schematic

A schematic is not just a stepping stone to the PCB. Someone else should be able to read it and understand your circuit without asking you questions. As you work, keep symbols logically grouped, label your nets clearly, and resist the temptation to cram everything into one corner. You'll thank yourself later when you're debugging.

#### The Symbol and Footprint Editors

KiCAD includes two additional editors you'll occasionally need to visit: the **Symbol Editor** and the **Footprint Editor**.

- The Symbol Editor lets you create or modify schematic symbols. Useful when a component isn't in the standard library.
- The Footprint Editor does the same for footprints, which are the physical pad layouts used in the PCB editor.

The built-in libraries and community resources cover the vast majority of common components. When you do need them, both editors follow the same general logic as the main tools.

#### Assigning Footprints

Every symbol in your schematic needs a corresponding **footprint** before you can move to the PCB editor. The footprint is the physical representation of a component: the copper pads, silkscreen outline, and courtyard boundary that will appear on your board.

You assign footprints either per-symbol in the component properties, or all at once using the **Footprint Assignment Tool** (`Tools → Assign Footprints`).

![](Module_8/Assets/6.png)

*Figure: Footprint Assignment Tool window showing symbol-to-footprint mapping.*

This is one of the steps where new designers most commonly get stuck. The key things to get right are:

| Parameter                          | Why it matters                                    |
| ---------------------------------- | ------------------------------------------------- |
| Package type (SMD vs through-hole) | Determines how the component is soldered          |
| Pad pitch and count                | Must match the physical component exactly         |
| Courtyard size                     | Prevents components from overlapping on the board |

When in doubt, cross-reference the component's datasheet. The mechanical drawing section will give you the exact pad dimensions and spacing you need.

#### Moving to the PCB Editor

Once your schematic is complete and footprints are assigned, you push the design into the PCB editor using `Tools → Update PCB from Schematic` (or press `F8`). This operation -- sometimes called a **forward annotation** -- transfers all your components and their connectivity information into the PCB workspace.

![](Module_8/Assets/8.png)

*Figure: PCB Editor immediately after forward annotation — components clustered in a pile with ratsnest lines visible. Image source: https://www.reddit.com/r/KiCad/comments/ry4zmu/kicad_6_and_messed_up_ratsnest/*

What you'll see at first is a pile of footprints clustered together, connected by thin lines called the **ratsnest**. The ratsnest is KiCAD's way of showing you which pads need to be connected — each line represents a net that doesn't yet have a physical copper trace. Your job in the PCB editor is to arrange the components and route those traces until the ratsnest disappears completely.

#### The PCB Editor

The PCB Editor is where your circuit becomes a physical object. It works in layers. Each layer represents something different about the final board.

| Layer                           | Purpose                                            |
| ------------------------------- | -------------------------------------------------- |
| `F.Cu` / `B.Cu`                 | Front and back copper — where your traces live     |
| `F.Silkscreen` / `B.Silkscreen` | Labels and outlines printed on the board surface   |
| `F.Courtyard` / `B.Courtyard`   | Keep-out zones around components                   |
| `Edge.Cuts`                     | The boundary of your board — defines its shape     |
| `F.Fab` / `B.Fab`               | Fabrication reference layer (not printed on board) |

For a simple two-layer board like the one I'm doing, you'll spend most of your time on `F.Cu`, `B.Cu`, and `Edge.Cuts`.

##### Placing Components

Before routing, you arrange your footprints on the board. There's no single right way to do this, but a few principles help:

- Components that are logically related should generally sit near each other. 
- Connectors and headers that a user will plug things into belong on the board's edges.
- Power components -- regulators, bulk capacitors -- should sit close to where power enters the board. 
- The goal is to minimise the distance traces need to travel, which reduces the complexity of the routing step.

##### Routing Traces

Routing is the process of replacing each ratsnest line with a real copper trace. You can route manually using the Route Track tool (`X`), or use the Interactive Router which intelligently avoids obstacles as you draw.

Trace width matters. A thin trace has more resistance and can carry less current before it heats up. For signal lines (I2C, UART, SPI) a default width of 0.25mm is typically fine. For power traces carrying any meaningful current, you'll want to go wider. A rough guide:

| Current    | Minimum trace width (1oz copper) |
| ---------- | -------------------------------- |
| < 500mA    | 0.25mm                           |
| 500mA – 1A | 0.5mm                            |
| 1A – 2A    | 1.0mm                            |
| 2A+        | 1.5mm or wider                   |

*Figure: PCB Editor with completed routing -- most ratsnest lines replaced by copper traces.*

##### Copper Pours

Rather than routing individual traces for power and ground, most designers flood one or both copper layers with a **copper pour** -- a solid region of copper assigned to a net, typically GND. Any pad connected to that net automatically gets a connection to the pour, which reduces the amount of manual routing and gives ground a low-impedance path across the whole board.

You create a copper pour using `Place → Add Filled Zone`, draw the boundary around your board, assign it to the GND net, and then press `B` to fill. KiCAD will automatically pour copper around all existing traces and pads.

#### Design Rule Check (DRC) and Electrical Rules Check (ERC)

Before sending your board for fabrication, KiCAD gives you two automated checks that catch the most common classes of mistake.

- The **Electrical Rules Check (ERC)** runs in the Schematic Editor. It looks for logical errors in your schematic: unconnected pins, pins driven by multiple sources, missing power flags, and so on. Run it from `Inspect → Electrical Rules Checker`. It won't catch every design mistake, but it will catch the obvious ones.

- The **Design Rule Check (DRC)** runs in the PCB Editor. It verifies that your physical layout meets the manufacturing constraints you've set -- minimum trace width, minimum clearance between copper features, unrouted nets, and so on. Run it from `Inspect → Design Rules Checker`.

![](Module_8/Assets/11.png)

*Figure: DRC results window showing violations list.*

Neither check guarantees your board will work; they only verify that it's logically consistent and manufacturable. But skipping them before fabrication is a reliable way to receive a board that doesn't behave as expected. Run both, resolve every violation, and only then move on.

#### Generating Gerber Files

When your board is routed, checked, and ready, you export it as a set of **Gerber files** -- the industry-standard format that PCB fabrication houses use to manufacture your board.

You generate Gerbers from `File → Fabrication Outputs → Gerbers`. KiCAD will ask you which layers to export. At minimum, you need:

- Front and back copper (`F.Cu`, `B.Cu`)
- Front and back silkscreen
- Front and back soldermask
- Board outline (`Edge.Cuts`)
- Drill file (generated separately via `File → Fabrication Outputs → Drill Files`)

Most fabrication houses (JLCPCB, PCBWay, OSH Park) accept a zipped folder of these files directly. Many also provide a KiCAD-specific guide on their website detailing exactly which layers to export and how to name them.

Find the JLCPCB Gerber upload guide here: https://jlcpcb.com/help/article/how-to-generate-gerber-and-drill-files-in-kicad-9
