
The MPU6050 is a 6-axis inertial measurement unit produced by InvenSense, now owned by TDK. It packages a 3-axis gyroscope and a 3-axis accelerometer into a single IC communicating over I2C. In robotics it is used for attitude estimation, balance control, vibration analysis, and motion detection. It is one of the most well-documented sensors available at this price point, and writing a driver for it from scratch is an excellent introduction to I2C peripheral configuration, register-level sensor initialisation, multi-byte burst reads, and fixed-point scaling arithmetic.

This chapter walks through the complete process from datasheet to working driver: understanding the register map, wiring the sensor to the STM32F4 Discovery board, configuring the I2C peripheral, initialising the device, reading raw data, applying the correct scaling, and verifying every stage with a logic analyzer.

## Section 1 -- Reading the Datasheet

Before writing a single line of code, spend time with the datasheet. This section walks through the parts that matter most for driver writing.

### 1.1 Device Overview and Pin Description

Open the datasheet and read Section 3 -- Product Overview. Note the following:

- The MPU6050 operates on a supply voltage of 2.375V to 3.46V. The STM32F4 Discovery's 3V3 rail is within this range and is the correct supply to use. Do not connect VCC to the 5V rail.

- The logic level on SDA and SCL is referenced to VLOGIC, which on most breakout boards is connected to VCC. This means the I2C lines operate at 3.3V logic, which is compatible with the STM32F4's GPIO without a level shifter.

- The AD0 pin selects between two I2C addresses. When AD0 is low the device address is 0x68. When AD0 is high the address is 0x69. This allows two MPU6050 devices on the same bus.

### 1.2 I2C Interface Characteristics

Navigate to the electrical characteristics section and find the I2C timing parameters. Note:

- The MPU6050 supports both standard mode I2C at 100kHz and fast mode at 400kHz. All driver examples in this chapter use 400kHz fast mode.

- SDA and SCL require pull-up resistors to VLOGIC. Most breakout boards include 4.7K ohm pull-ups onboard. If you are using a bare IC you must add them externally. Check your specific breakout board schematic before proceeding.

### 1.3 Register Map Overview

Open the register map document. This is the document you will use most heavily. The key registers for a basic driver are:

**PWR_MGMT_1** at address 0x6B controls power state and clock source. On power-up the device is in sleep mode and must be woken before any measurements can be taken.

**SMPLRT_DIV** at address 0x19 sets the sample rate divider. The output sample rate is determined by dividing the gyroscope output rate by this value plus one.

**CONFIG** at address 0x1A sets the digital low-pass filter bandwidth for both the gyroscope and accelerometer.

**GYRO_CONFIG** at address 0x1B sets the gyroscope full-scale range.

**ACCEL_CONFIG** at address 0x1C sets the accelerometer full-scale range.

**ACCEL_XOUT_H** through GYRO_ZOUT_L at addresses 0x3B through 0x48 contain the 14 bytes of raw sensor data -- 6 bytes of accelerometer data, 2 bytes of temperature data, and 6 bytes of gyroscope data. These 14 bytes are read in a single burst read during normal operation.

**WHO_AM_I** at address 0x75 contains the device identity byte. On the MPU6050 this always reads 0x68 regardless of the AD0 pin state. Reading this register is the standard way to confirm the device is present and responding on the bus before proceeding with initialisation.

### 1.4 Sensitivity and Full-Scale Range

Navigate to Section 3 of the register map covering GYRO_CONFIG and ACCEL_CONFIG.

The gyroscope full-scale range is selected by bits FS_SEL[1:0] in register 0x1B. The four options are plus or minus 250, 500, 1000, or 2000 degrees per second. The corresponding sensitivity scale factor -- how many LSB per degree per second the raw output represents -- is 131, 65.5, 32.8, or 16.4 respectively.

The accelerometer full-scale range is selected by bits AFS_SEL[1:0] in register 0x1C. The four options are plus or minus 2, 4, 8, or 16g. The corresponding sensitivity scale factor is 16384, 8192, 4096, or 2048 LSB per g respectively.

Write these numbers down. You will use them in the scaling calculations in Section 5.

## Section 2 -- Wiring

Connect the MPU6050 breakout board to the STM32F4 Discovery board as follows:

- MPU6050 VCC connects to Discovery 3V3
- MPU6050 GND connects to Discovery GND 
- MPU6050 SDA connects to Discovery PB7
- MPU6050 SCL connects to Discovery PB6
- MPU6050 AD0 connects to Discovery GND (sets I2C address to 0x68)
- MPU6050 INT is left unconnected for this chapter

PB6 and PB7 are the default I2C1 pins on the STM32F4 Discovery and are broken out on the board's expansion headers.

Logic analyzer connections for this chapter:

- Connect logic analyzer channel 1 to PB7 (SDA)

## Section 4 -- Driver Implementation

Create two new files in your project: MPU6050.h in Core/Inc and MPU6050.c in Core/Src.

### 4.1 MPU6050.h

```c
#ifndef MPU6050_H
#define MPU6050_H

#include "stm32f4xx_hal.h"
#include <stdint.h>

// I2C address (AD0 low = 0x68, shifted left 1 for HAL)
#define MPU6050_ADDR            (0x68 << 1)

// Register addresses
#define MPU6050_REG_SMPLRT_DIV  0x19
#define MPU6050_REG_CONFIG      0x1A
#define MPU6050_REG_GYRO_CONFIG 0x1B
#define MPU6050_REG_ACCEL_CONFIG 0x1C
#define MPU6050_REG_ACCEL_XOUT_H 0x3B
#define MPU6050_REG_TEMP_OUT_H  0x41
#define MPU6050_REG_GYRO_XOUT_H 0x43
#define MPU6050_REG_PWR_MGMT_1  0x6B
#define MPU6050_REG_WHO_AM_I    0x75

// Gyroscope full-scale range options
typedef enum {
    GYRO_FS_250  = 0x00,   // +/- 250 deg/s  -- 131 LSB/deg/s
    GYRO_FS_500  = 0x08,   // +/- 500 deg/s  -- 65.5 LSB/deg/s
    GYRO_FS_1000 = 0x10,   // +/- 1000 deg/s -- 32.8 LSB/deg/s
    GYRO_FS_2000 = 0x18    // +/- 2000 deg/s -- 16.4 LSB/deg/s
} MPU6050_GyroFS;

// Accelerometer full-scale range options
typedef enum {
    ACCEL_FS_2G  = 0x00,   // +/- 2g  -- 16384 LSB/g
    ACCEL_FS_4G  = 0x08,   // +/- 4g  -- 8192 LSB/g
    ACCEL_FS_8G  = 0x10,   // +/- 8g  -- 4096 LSB/g
    ACCEL_FS_16G = 0x18    // +/- 16g -- 2048 LSB/g
} MPU6050_AccelFS;

// Scaled output structure
typedef struct {
    float accel_x;   // g
    float accel_y;   // g
    float accel_z;   // g
    float gyro_x;    // degrees per second
    float gyro_y;    // degrees per second
    float gyro_z;    // degrees per second
    float temp_c;    // degrees Celsius
} MPU6050_Data;

// Driver handle
typedef struct {
    I2C_HandleTypeDef *hi2c;
    MPU6050_GyroFS     gyro_fs;
    MPU6050_AccelFS    accel_fs;
    float              gyro_sensitivity;    // LSB per deg/s
    float              accel_sensitivity;   // LSB per g
} MPU6050_Handle;

// Function prototypes
HAL_StatusTypeDef MPU6050_Init(MPU6050_Handle *dev,
                                I2C_HandleTypeDef *hi2c,
                                MPU6050_GyroFS gyro_fs,
                                MPU6050_AccelFS accel_fs);

HAL_StatusTypeDef MPU6050_ReadRaw(MPU6050_Handle *dev,
                                   int16_t *accel,
                                   int16_t *gyro,
                                   int16_t *temp);

HAL_StatusTypeDef MPU6050_ReadScaled(MPU6050_Handle *dev,
                                      MPU6050_Data *out);

#endif
```

### 4.2 MPU6050.c

```c
#include "MPU6050.h"

// Write a single byte to a register
static HAL_StatusTypeDef write_reg(MPU6050_Handle *dev,
                                    uint8_t reg,
                                    uint8_t val) {
    uint8_t buf[2] = { reg, val };
    return HAL_I2C_Master_Transmit(dev->hi2c, MPU6050_ADDR,
                                    buf, 2, HAL_MAX_DELAY);
}

// Read one or more bytes starting from a register
static HAL_StatusTypeDef read_reg(MPU6050_Handle *dev,
                                   uint8_t reg,
                                   uint8_t *buf,
                                   uint16_t len) {
    HAL_StatusTypeDef s;
    s = HAL_I2C_Master_Transmit(dev->hi2c, MPU6050_ADDR,
                                  &reg, 1, HAL_MAX_DELAY);
    if (s != HAL_OK) return s;
    return HAL_I2C_Master_Receive(dev->hi2c, MPU6050_ADDR,
                                   buf, len, HAL_MAX_DELAY);
}

HAL_StatusTypeDef MPU6050_Init(MPU6050_Handle *dev,
                                I2C_HandleTypeDef *hi2c,
                                MPU6050_GyroFS gyro_fs,
                                MPU6050_AccelFS accel_fs) {
    dev->hi2c     = hi2c;
    dev->gyro_fs  = gyro_fs;
    dev->accel_fs = accel_fs;
    HAL_StatusTypeDef s;

    // Step 1 -- Verify device identity
    uint8_t who_am_i = 0;
    s = read_reg(dev, MPU6050_REG_WHO_AM_I, &who_am_i, 1);
    if (s != HAL_OK || who_am_i != 0x68) return HAL_ERROR;

    // Step 2 -- Wake the device and select PLL with X-axis gyro as clock
    // Bit 6 (SLEEP) = 0 to exit sleep mode
    // Bits 2:0 (CLKSEL) = 001 for PLL with X-axis gyro reference
    // Using the gyro PLL gives better clock stability than the internal oscillator
    s = write_reg(dev, MPU6050_REG_PWR_MGMT_1, 0x01);
    if (s != HAL_OK) return s;
    HAL_Delay(100);   // Allow PLL to stabilise

    // Step 3 -- Set sample rate divider
    // Sample rate = Gyro output rate / (1 + SMPLRT_DIV)
    // Gyro output rate is 1kHz when DLPF is enabled
    // 0x07 gives 1000 / (1 + 7) = 125Hz sample rate
    s = write_reg(dev, MPU6050_REG_SMPLRT_DIV, 0x07);
    if (s != HAL_OK) return s;

    // Step 4 -- Configure digital low-pass filter
    // CONFIG bits 2:0 (DLPF_CFG) = 0x01
    // Accelerometer: 184Hz bandwidth, 2.0ms delay
    // Gyroscope:     188Hz bandwidth, 1.9ms delay
    s = write_reg(dev, MPU6050_REG_CONFIG, 0x01);
    if (s != HAL_OK) return s;

    // Step 5 -- Set gyroscope full-scale range
    s = write_reg(dev, MPU6050_REG_GYRO_CONFIG, (uint8_t)gyro_fs);
    if (s != HAL_OK) return s;

    // Step 6 -- Set accelerometer full-scale range
    s = write_reg(dev, MPU6050_REG_ACCEL_CONFIG, (uint8_t)accel_fs);
    if (s != HAL_OK) return s;

    // Step 7 -- Store sensitivity scale factors for use in scaling calculations
    // Values taken from register map document, Section 3
    switch (gyro_fs) {
        case GYRO_FS_250:  dev->gyro_sensitivity = 131.0f;  break;
        case GYRO_FS_500:  dev->gyro_sensitivity = 65.5f;   break;
        case GYRO_FS_1000: dev->gyro_sensitivity = 32.8f;   break;
        case GYRO_FS_2000: dev->gyro_sensitivity = 16.4f;   break;
    }

    switch (accel_fs) {
        case ACCEL_FS_2G:  dev->accel_sensitivity = 16384.0f; break;
        case ACCEL_FS_4G:  dev->accel_sensitivity = 8192.0f;  break;
        case ACCEL_FS_8G:  dev->accel_sensitivity = 4096.0f;  break;
        case ACCEL_FS_16G: dev->accel_sensitivity = 2048.0f;  break;
    }

    return HAL_OK;
}

HAL_StatusTypeDef MPU6050_ReadRaw(MPU6050_Handle *dev,
                                   int16_t *accel,
                                   int16_t *gyro,
                                   int16_t *temp) {
    // Burst read all 14 bytes starting at ACCEL_XOUT_H
    // Layout: AX_H AX_L AY_H AY_L AZ_H AZ_L
    //         TEMP_H TEMP_L
    //         GX_H GX_L GY_H GY_L GZ_H GZ_L
    uint8_t buf[14];
    HAL_StatusTypeDef s = read_reg(dev,
                                    MPU6050_REG_ACCEL_XOUT_H,
                                    buf, 14);
    if (s != HAL_OK) return s;

    accel[0] = (int16_t)(buf[0]  << 8 | buf[1]);
    accel[1] = (int16_t)(buf[2]  << 8 | buf[3]);
    accel[2] = (int16_t)(buf[4]  << 8 | buf[5]);
    *temp    = (int16_t)(buf[6]  << 8 | buf[7]);
    gyro[0]  = (int16_t)(buf[8]  << 8 | buf[9]);
    gyro[1]  = (int16_t)(buf[10] << 8 | buf[11]);
    gyro[2]  = (int16_t)(buf[12] << 8 | buf[13]);

    return HAL_OK;
}

HAL_StatusTypeDef MPU6050_ReadScaled(MPU6050_Handle *dev,
                                      MPU6050_Data *out) {
    int16_t accel[3], gyro[3], temp;
    HAL_StatusTypeDef s = MPU6050_ReadRaw(dev, accel, gyro, &temp);
    if (s != HAL_OK) return s;

    // Accelerometer: divide raw value by sensitivity to get g
    out->accel_x = accel[0] / dev->accel_sensitivity;
    out->accel_y = accel[1] / dev->accel_sensitivity;
    out->accel_z = accel[2] / dev->accel_sensitivity;

    // Gyroscope: divide raw value by sensitivity to get degrees per second
    out->gyro_x  = gyro[0]  / dev->gyro_sensitivity;
    out->gyro_y  = gyro[1]  / dev->gyro_sensitivity;
    out->gyro_z  = gyro[2]  / dev->gyro_sensitivity;

    // Temperature: formula from register map Section 4.18
    // Temperature in degrees C = (TEMP_OUT / 340.0) + 36.53
    out->temp_c  = (temp / 340.0f) + 36.53f;

    return HAL_OK;
}
```

## Section 5 -- Mathematical Manipulation

### 5.1 From Raw ADC Value to Physical Unit

The MPU6050 outputs signed 16-bit integers for every axis. These raw values have no physical meaning on their own -- they represent a fraction of the selected full-scale range. Converting them to physical units requires dividing by the sensitivity scale factor for the selected range.

For the accelerometer at plus or minus 2g:

```
Acceleration (g) = Raw ADC value / 16384
```

If the raw X accelerometer value is 8192, the acceleration is 8192 / 16384 = 0.5g.

If the sensor is lying flat and stationary, the Z axis should read approximately 16384 -- representing 1g of gravitational acceleration. The X and Y axes should read approximately 0.

For the gyroscope at plus or minus 250 degrees per second:

```
Angular rate (deg/s) = Raw ADC value / 131
```

If the raw Z gyroscope value is 3277, the angular rate is 3277 / 131 = 25.0 degrees per second.

For temperature:

```
Temperature (C) = (Raw TEMP_OUT / 340.0) + 36.53
```

This formula is taken directly from Section 4.18 of the register map document. The constants 340.0 and 36.53 are fixed device characteristics, not configuration-dependent.

### 5.2 Complementary Filter for Tilt Angle

Raw accelerometer and gyroscope data each have limitations when used alone to estimate tilt angle. The accelerometer gives an accurate long-term tilt estimate but is noisy and sensitive to vibration. The gyroscope integrates angular rate to track fast motion but accumulates drift over time.

A complementary filter combines both to get a stable, responsive tilt estimate:

```c
#define ALPHA       0.98f    // Trust gyro for fast motion
#define DT          0.008f   // Sample period in seconds (125Hz = 8ms)

float angle_x = 0.0f;
float angle_y = 0.0f;

// Inside your read loop:

// Accelerometer tilt angle in degrees
float accel_angle_x = atan2f(data.accel_y, data.accel_z) * (180.0f / M_PI);
float accel_angle_y = atan2f(-data.accel_x, data.accel_z) * (180.0f / M_PI);

// Complementary filter: blend gyro integration with accel correction
angle_x = ALPHA * (angle_x + data.gyro_x * DT) + (1.0f - ALPHA) * accel_angle_x;
angle_y = ALPHA * (angle_y + data.gyro_y * DT) + (1.0f - ALPHA) * accel_angle_y;
```

ALPHA controls the blend ratio. A value of 0.98 means 98 percent of the angle estimate comes from the integrated gyroscope and 2 percent from the accelerometer correction. This gives responsive tracking of fast motion while the accelerometer prevents long-term drift.

DT must match your actual sample period. At 125Hz the period is 8ms or 0.008 seconds. If you change SMPLRT_DIV you must update DT accordingly.

```c
/* USER CODE BEGIN WHILE */
while (1) {

    MPU6050_Data data;
    if (MPU6050_ReadScaled(&mpu, &data) == HAL_OK) {

        // Complementary filter tilt angles
        float accel_angle_x = atan2f(data.accel_y, data.accel_z)
                               * (180.0f / M_PI);
        float accel_angle_y = atan2f(-data.accel_x, data.accel_z)
                               * (180.0f / M_PI);

        angle_x = ALPHA * (angle_x + data.gyro_x * DT)
                  + (1.0f - ALPHA) * accel_angle_x;
        angle_y = ALPHA * (angle_y + data.gyro_y * DT)
                  + (1.0f - ALPHA) * accel_angle_y;

        printf("Accel(g)  X:%6.3f Y:%6.3f Z:%6.3f  "
               "Gyro(d/s) X:%7.3f Y:%7.3f Z:%7.3f  "
               "Tilt X:%6.2f Y:%6.2f  Temp:%5.2fC\r\n",
               data.accel_x, data.accel_y, data.accel_z,
               data.gyro_x,  data.gyro_y,  data.gyro_z,
               angle_x, angle_y, data.temp_c);
    }

    HAL_Delay(8);   // 125Hz
}
/* USER CODE END WHILE */
```

## Section 7 -- Verifying with a Logic Analyzer

Connect your logic analyzer to PB6 (SCL) and PB7 (SDA) as described in Section 2. Open PulseView, add an I2C decoder, assign SCL to channel 0 and SDA to channel 1, and set the sample rate to at least 2MHz.
### 7.1 Verifying the WHO_AM_I Read

Trigger a capture and look for the first I2C transaction after reset. You should see:

A start condition, followed by the 7-bit address 0x68 with a write bit, followed by the register address byte 0x75. Then a repeated start, the address 0x68 with a read bit, and one data byte reading 0x68. A stop condition follows.

This is the WHO_AM_I read inside MPU6050_Init. Confirming this transaction tells you the I2C bus is working, the device is present at the correct address, and the first register read is returning the expected identity value.

![[Pasted image 20260413013704.png]]
*Figure: Signal Analysis of the I2C Line. Note the start of the communication between the MPU and the STM, as described above.* 

### 7.2 Verifying the Initialisation Sequence

After the WHO_AM_I read you should see a series of write transactions corresponding to each register write in MPU6050_Init. Verify each one:

- Address 0x68 write to register 0x6B with value 0x01 -- PWR_MGMT_1, wake and set clock. 
- Address 0x68 write to register 0x19 with value 0x07 -- SMPLRT_DIV, 125Hz. 
- Address 0x68 write to register 0x1A with value 0x01 -- CONFIG, DLPF enabled. 
- Address 0x68 write to register 0x1B with value 0x00 -- GYRO_CONFIG, plus or minus 250 deg/s. 
- Address 0x68 write to register 0x1C with value 0x00 -- ACCEL_CONFIG, plus or minus 2g.

Cross-reference each transaction against the register map document to confirm the correct register is being written with the correct value.

### 7.3 Verifying the Burst Read

Each call to MPU6050_ReadRaw generates one burst read transaction. You should see:

A write to register 0x3B to set the read pointer, then a repeated start and a 14-byte read returning all accelerometer, temperature, and gyroscope data in sequence. The I2C decoder in PulseView will display all 14 bytes individually.

Confirm the transaction is 14 bytes long. Confirm the first byte address is 0x3B. If you rotate the sensor while the capture is running you should see the raw byte values changing between captures.

## What You Should Take Away

The WHO_AM_I register is always the first register you read when initialising any sensor. If it does not return the expected value, stop and check your wiring, supply voltage, and I2C address before proceeding. The sleep bit in PWR_MGMT_1 is the most common reason a freshly wired MPU6050 appears to be present but returns zeros for all sensor data. Raw ADC values have no physical meaning without dividing by the correct sensitivity scale factor for the configured full-scale range, and that factor changes every time you change the range. Burst reading all 14 bytes in a single I2C transaction is always preferable to reading each register individually -- it is faster, produces a consistent snapshot of all axes at the same moment, and reduces bus traffic. The complementary filter is the simplest practical approach to tilt estimation from a 6-axis IMU, and understanding its two input paths and the role of ALPHA builds the intuition needed for more advanced filters like the Kalman filter.
