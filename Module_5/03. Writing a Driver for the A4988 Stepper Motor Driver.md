The A4988 is a bipolar stepper motor driver produced by Allegro MicroSystems. Unlike the sensors covered in this content, the A4988 is an actuator driver -- it does not communicate over I2C or SPI. Instead it exposes a minimal step-and-direction interface to the microcontroller: one GPIO pin to set rotation direction and one GPIO pin to receive step pulses. Each rising edge on the STEP pin advances the motor by one step or microstep depending on the microstepping configuration. All current control, coil switching, and microstepping logic is handled entirely inside the A4988 chip.

This makes the A4988 an excellent device for learning hardware timer-driven pulse generation on the STM32F4. The driver has no complex register map and no communication protocol -- the entire challenge is generating precise, correctly timed pulses using the STM32's timer peripheral, and building a velocity profile that accelerates and decelerates the motor smoothly rather than stepping at a fixed rate.

## Section 1 -- Reading the Datasheet

### 1.1 Device Overview

Open the datasheet and read the product description. Note the following:

The A4988 drives bipolar stepper motors with output current up to 2A per coil. It requires two supply voltages -- a logic supply VDD between 3V and 5.5V, and a motor supply VMOT between 8V and 35V. Since no motor is used in this chapter only VDD needs to be connected. VMOT and the motor outputs are left unconnected.

The device has five control inputs relevant to this driver: STEP, DIR, MS1, MS2, MS3, and ENABLE. All are logic-level inputs compatible with 3.3V from the STM32F4.

![[Pasted image 20260413141616.png]]
*Figure: A4988 datasheet pin diagram and functional block diagram.*

### 1.2 Step and Direction Interface

The core of the A4988 interface is straightforward. The DIR pin sets rotation direction -- high for one direction, low for the other. The STEP pin advances the motor by one step increment on each rising edge. The minimum pulse width requirements from the datasheet electrical characteristics table are:

STEP high time minimum: 1 microsecond STEP low time minimum: 1 microsecond DIR setup time before STEP rising edge: 200 nanoseconds

These timing constraints define the maximum step rate. At a minimum pulse width of 1 microsecond high and 1 microsecond low, the theoretical maximum step rate is 500kHz. In practice you will operate far below this -- typical stepper motors cannot mechanically respond above a few thousand steps per second.

![[Pasted image 20260413141724.png]]
*Figure: A4988 datasheet timing diagram showing STEP and DIR signal requirements*

### 1.3 Microstepping Configuration

The MS1, MS2, and MS3 pins select the microstepping resolution. The truth table from the datasheet is:

- MS1 low, MS2 low, MS3 low: full step 
- MS1 high, MS2 low, MS3 low: half step
- MS1 low, MS2 high, MS3 low: quarter step 
- MS1 high, MS2 high, MS3 low: eighth step 
- MS1 high, MS2 high, MS3 high: sixteenth step

Microstepping divides each full motor step into smaller increments, producing smoother motion and finer positioning resolution at the cost of requiring more pulses per revolution. A standard stepper motor has 200 full steps per revolution. At sixteenth microstepping that becomes 3200 microsteps per revolution.

Most A4988 breakout boards have pull-down resistors on MS1, MS2, and MS3, defaulting to full step mode when the pins are left unconnected. This chapter drives all three from STM32 GPIO so the microstepping mode can be changed in software.

![[Pasted image 20260413141410.png]]
*Figure: A4988 microstepping truth table from datasheet.*

### 1.4 ENABLE Pin

The ENABLE pin is active low. When driven low the A4988 output drivers are enabled and the motor coils are energised. When driven high the outputs are disabled and the motor coils are de-energised. On most breakout boards ENABLE has a pull-down resistor, meaning the outputs are enabled by default when the pin is left unconnected. This chapter drives ENABLE explicitly from a GPIO pin for full software control.

### 3.2 Configure TIM2 for PWM Output

The STEP signal is generated by TIM2 Channel 1 in PWM mode. This allows the STM32 hardware timer to produce pulses autonomously without CPU intervention, freeing the main loop to handle acceleration calculations.

- Open the .ioc file and navigate to Timers, TIM2.

- Set Clock Source to Internal Clock. Set Channel 1 to PWM Generation CH1.

- Under Parameter Settings set the following: Prescaler to 83. This divides the 84MHz APB1 timer clock down to 1MHz, giving a timer tick period of 1 microsecond. Counter Period to 999. This sets the PWM period to 1000 ticks, giving a base frequency of 1kHz. This will be changed dynamically by the driver. Pulse to 500. This sets the initial duty cycle to 50 percent. Counter Mode to Up. Auto-Reload Preload to Enable.

- Under NVIC Settings enable TIM2 global interrupt. This is used by the acceleration profile to update the timer period on each step.

- Click on PA0, and set it to TIM2_CH1.

### 4.1 A4988.h

```c

#define A4988_H

#include "stm32f4xx_hal.h"
#include <stdint.h>

// Microstepping modes
typedef enum {
    STEP_FULL      = 0,   // MS1=0 MS2=0 MS3=0
    STEP_HALF      = 1,   // MS1=1 MS2=0 MS3=0
    STEP_QUARTER   = 2,   // MS1=0 MS2=1 MS3=0
    STEP_EIGHTH    = 3,   // MS1=1 MS2=1 MS3=0
    STEP_SIXTEENTH = 4    // MS1=1 MS2=1 MS3=1
} A4988_MicrostepMode;

// Rotation direction
typedef enum {
    DIR_CW  = 0,
    DIR_CCW = 1
} A4988_Direction;

// Driver handle
typedef struct {
    TIM_HandleTypeDef *htim;         // Timer used for STEP pulse generation
    uint32_t           tim_channel;  // Timer channel (e.g. TIM_CHANNEL_1)

    GPIO_TypeDef      *dir_port;
    uint16_t           dir_pin;

    GPIO_TypeDef      *enable_port;
    uint16_t           enable_pin;

    GPIO_TypeDef      *ms1_port;
    uint16_t           ms1_pin;

    GPIO_TypeDef      *ms2_port;
    uint16_t           ms2_pin;

    GPIO_TypeDef      *ms3_port;
    uint16_t           ms3_pin;

    // Motion state
    uint32_t  steps_remaining;   // Steps left in current move
    uint32_t  steps_total;       // Total steps in current move
    uint32_t  accel_steps;       // Number of steps spent accelerating
    uint32_t  current_period;    // Current timer ARR value
    uint32_t  min_period;        // ARR at maximum speed
    uint32_t  max_period;        // ARR at minimum speed (start speed)
    uint8_t   running;           // 1 if a move is in progress
} A4988_Handle;

// Function prototypes
void A4988_Init(A4988_Handle *dev,
                TIM_HandleTypeDef *htim,
```

### 4.2 A4988.c

```c
#include "A4988.h"

void A4988_Init(A4988_Handle *dev,
                TIM_HandleTypeDef *htim,
                uint32_t channel) {
    dev->htim        = htim;
    dev->tim_channel = channel;

    // Wire GPIO ports and pins to match your IOC configuration
    dev->dir_port    = GPIOA; dev->dir_pin    = GPIO_PIN_1;
    dev->enable_port = GPIOE; dev->enable_pin = GPIO_PIN_7;
    dev->ms1_port    = GPIOE; dev->ms1_pin    = GPIO_PIN_8;
    dev->ms2_port    = GPIOE; dev->ms2_pin    = GPIO_PIN_9;
    dev->ms3_port    = GPIOE; dev->ms3_pin    = GPIO_PIN_10;

    dev->running = 0;

    // Default state: disabled, full step, clockwise
    A4988_Disable(dev);
    A4988_SetMicrostep(dev, STEP_FULL);
    A4988_SetDirection(dev, DIR_CW);
}

void A4988_SetMicrostep(A4988_Handle *dev, A4988_MicrostepMode mode) {
    // Set MS1, MS2, MS3 according to truth table in datasheet
    uint8_t ms1, ms2, ms3;

    switch (mode) {
        case STEP_FULL:      ms1=0; ms2=0; ms3=0; break;
        case STEP_HALF:      ms1=1; ms2=0; ms3=0; break;
        case STEP_QUARTER:   ms1=0; ms2=1; ms3=0; break;
        case STEP_EIGHTH:    ms1=1; ms2=1; ms3=0; break;
        case STEP_SIXTEENTH: ms1=1; ms2=1; ms3=1; break;
        default:             ms1=0; ms2=0; ms3=0; break;
    }

    HAL_GPIO_WritePin(dev->ms1_port, dev->ms1_pin,
                      ms1 ? GPIO_PIN_SET : GPIO_PIN_RESET);
    HAL_GPIO_WritePin(dev->ms2_port, dev->ms2_pin,
                      ms2 ? GPIO_PIN_SET : GPIO_PIN_RESET);
    HAL_GPIO_WritePin(dev->ms3_port, dev->ms3_pin,
                      ms3 ? GPIO_PIN_SET : GPIO_PIN_RESET);
}

void A4988_SetDirection(A4988_Handle *dev, A4988_Direction dir) {
    HAL_GPIO_WritePin(dev->dir_port, dev->dir_pin,
                      dir == DIR_CW ? GPIO_PIN_RESET : GPIO_PIN_SET);
}

void A4988_Enable(A4988_Handle *dev) {
    // ENABLE is active low
    HAL_GPIO_WritePin(dev->enable_port, dev->enable_pin, GPIO_PIN_RESET);
}

void A4988_Disable(A4988_Handle *dev) {
    HAL_GPIO_WritePin(dev->enable_port, dev->enable_pin, GPIO_PIN_SET);
}

void A4988_MoveSteps(A4988_Handle *dev,
                     uint32_t steps,
                     uint32_t min_period,
                     uint32_t max_period,
                     uint32_t accel_steps) {
    if (steps == 0) return;

    dev->steps_remaining = steps;
    dev->steps_total     = steps;
    dev->accel_steps     = accel_steps;
    dev->min_period      = min_period;
    dev->max_period      = max_period;
    dev->current_period  = max_period;
    dev->running         = 1;

    __HAL_TIM_SET_AUTORELOAD(dev->htim, max_period);
    __HAL_TIM_SET_COMPARE(dev->htim, dev->tim_channel, max_period / 2);

    // Start PWM output on the channel (generates the STEP signal)
    HAL_TIM_PWM_Start(dev->htim, dev->tim_channel);
    
    __HAL_TIM_CLEAR_FLAG(dev->htim, TIM_FLAG_UPDATE);
    __HAL_TIM_SET_COUNTER(dev->htim, 0); HAL_TIM_PWM_Start(dev->htim,
     dev->tim_channel); HAL_TIM_Base_Start_IT(dev->htim);

    // Start the base timer interrupt separately (fires on period reload)
    HAL_TIM_Base_Start_IT(dev->htim);
}

void A4988_StepCallback(A4988_Handle *dev) {
    if (!dev->running) return;

    if (dev->steps_remaining == 0) {
        HAL_TIM_PWM_Stop(dev->htim, dev->tim_channel);
        HAL_TIM_Base_Stop_IT(dev->htim);
        dev->running = 0;
        return;
    }

    dev->steps_remaining--;

    if (dev->steps_remaining == 0) {
        HAL_TIM_PWM_Stop(dev->htim, dev->tim_channel);
        HAL_TIM_Base_Stop_IT(dev->htim);
        dev->running = 0;
        return;
    }
    
    // Trapezoidal velocity profile
    // Steps completed so far
    uint32_t steps_done = dev->steps_total - dev->steps_remaining;

    // Deceleration starts this many steps before the end
    uint32_t decel_start = dev->steps_total - dev->accel_steps;

    if (steps_done < dev->accel_steps) {
        // Acceleration phase -- decrease period (increase speed)
        // Linear interpolation from max_period down to min_period
        uint32_t period_range = dev->max_period - dev->min_period;
        dev->current_period = dev->max_period
                              - (period_range * steps_done)
                              / dev->accel_steps;
    } else if (steps_done >= decel_start) {
        // Deceleration phase -- increase period (decrease speed)
        uint32_t steps_in_decel = steps_done - decel_start;
        uint32_t period_range   = dev->max_period - dev->min_period;
        dev->current_period = dev->min_period
                              + (period_range * steps_in_decel)
                              / dev->accel_steps;
    } else {
        // Cruise phase -- hold at minimum period (maximum speed)
        dev->current_period = dev->min_period;
    }

    // Clamp to valid range
    if (dev->current_period < dev->min_period)
        dev->current_period = dev->min_period;
    if (dev->current_period > dev->max_period)
        dev->current_period = dev->max_period;

    // Update timer period and pulse width
    __HAL_TIM_SET_AUTORELOAD(dev->htim, dev->current_period);
    __HAL_TIM_SET_COMPARE(dev->htim, dev->tim_channel,
                           dev->current_period / 2);
}

uint8_t A4988_IsRunning(A4988_Handle *dev) {
    return dev->running;
}
```


## Section 5 -- Mathematical Manipulation

### 5.1 Timer Period to Step Frequency

The STM32 timer is configured with a prescaler of 83, dividing the 84MHz APB1 clock to 1MHz. Each timer tick is therefore 1 microsecond. The Auto-Reload Register (ARR) value sets the period of the PWM waveform in timer ticks.

The relationship between ARR and step frequency is:

```
Step frequency (Hz) = Timer clock (Hz) / ARR
                    = 1,000,000 / ARR
```

So an ARR of 1000 gives 1000 steps per second. An ARR of 500 gives 2000 steps per second. An ARR of 100 gives 10000 steps per second.

To calculate the ARR for a desired step frequency:

```
ARR = Timer clock / Desired step frequency
    = 1,000,000 / Step frequency
```

### 5.2 Steps Per Revolution and RPM

A standard stepper motor has 200 full steps per revolution (1.8 degrees per step). The number of steps per revolution at a given microstepping mode is:

```
Steps per revolution = 200 * Microstep multiplier
```

At full step: 200 steps per revolution At half step: 400 steps per revolution At quarter step: 800 steps per revolution At eighth step: 1600 steps per revolution At sixteenth step: 3200 steps per revolution

RPM is calculated from step frequency and steps per revolution:

```
RPM = (Step frequency * 60) / Steps per revolution
```

At 1000 steps per second in full step mode: RPM = (1000 * 60) / 200 = 300 RPM

At 1000 steps per second in sixteenth step mode: RPM = (1000 * 60) / 3200 = 18.75 RPM

### 5.3 Trapezoidal Velocity Profile

Stepping at a constant rate from standstill causes mechanical resonance and missed steps because the motor rotor cannot accelerate instantaneously. A trapezoidal velocity profile ramps the step frequency up from a low starting speed, holds at maximum speed for the cruise phase, then ramps back down before stopping.

The profile has three phases. In the acceleration phase the timer period decreases linearly from max_period down to min_period over accel_steps steps. In the cruise phase the timer holds at min_period. In the deceleration phase the period increases linearly from min_period back to max_period over accel_steps steps.

```
Speed
  |          ___________
  |         /           \
  |        /             \
  |_______/               \________
  |
  +--accel--+----cruise---+-decel---> Steps
```

The period at any step during acceleration is:

```
period = max_period - ((max_period - min_period) * steps_done / accel_steps)
```

The period at any step during deceleration is:

```
period = min_period + ((max_period - min_period) * steps_in_decel / accel_steps)
```

These calculations happen inside A4988_StepCallback on every timer interrupt, updating the ARR register in real time as the move progresses.

### 5.4 Choosing min_period, max_period and accel_steps

max_period sets the starting and ending speed. A value of 2000 (500Hz) is a safe starting speed for most small stepper motors in full step mode.

min_period sets the maximum speed. A value of 200 (5000Hz) corresponds to 1500 RPM in full step mode. Start conservatively and reduce min_period gradually.

accel_steps controls how quickly the profile accelerates. A longer ramp produces smoother acceleration but takes more steps to reach full speed. A starting value of 100 steps is reasonable for most motors.

## What You Should Take Away

The A4988's step-and-direction interface moves the complexity from the communication protocol layer -- which does not exist here -- into the timer peripheral configuration and the motion profile algorithm. Hardware PWM generation via the STM32 timer produces step pulses with microsecond-level precision that software GPIO toggling cannot match. The trapezoidal velocity profile is not just a refinement -- at any non-trivial step rate, starting at full speed will cause a real motor to stall or lose steps because the rotor cannot accelerate instantaneously. The timer period is the fundamental control variable: reducing it increases step frequency and motor speed, and the entire motion profile reduces to updating this one register value on every step interrupt. Verifying the profile shape on the logic analyzer before connecting a motor is good engineering practice -- it confirms the timing is correct before any mechanical load is involved.
